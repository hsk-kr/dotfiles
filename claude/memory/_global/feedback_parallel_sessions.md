---
name: Respect parallel session boundaries
description: User runs parallel Claude sessions — always ask what's being handled elsewhere before starting work
type: feedback
---

Before starting work on a feature area, ask if the user is handling any part of it in another session.

**Why:** User often splits work across sessions. Duplicating work wastes time and causes conflicts.

**How to apply:** When a task involves both code infrastructure AND content/data, assume the user may be splitting the work. Ask before generating content or data.
