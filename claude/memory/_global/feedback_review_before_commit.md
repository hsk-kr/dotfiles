Always review code before creating a commit. This is a consistent ritual the user follows.

**Why:** The user says "review your code once more" or "review the code please" before almost every commit. It's a quality gate they rely on.

**How to apply:** Before any `git commit`, re-read the changed files and check for bugs, style issues, missing edge cases. If you find issues, fix them before committing. Don't wait to be asked.
